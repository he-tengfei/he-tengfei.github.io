七、面向对象上

OOP = Object Oriented Programming，面向对象的程序设计。

1. 类与对象

1.1 为什么需要类与对象？

用面向对象的方式就完美解决了上述两种方式存在的问题。

1.2 类与对象的关系

  • 类就是数据类型，比如上面的Cat，是一种自定义数据类型，可以自定义里面都有哪些属性和方法
  • 而对象就是一个类的具体实例，比如上面的Cat就是猫这个类，而小花这只猫就是Cat类的一个对象
  • 从类到对象有几种不同的说法，比如创建一个对象、把类实例化等，说的都是同一个意思
  • 类是抽象的，代表一类事物，比如猫；对象是具体的，代表一个具体事物，比如小花这只猫

1.4 属性和成员变量

这俩极其相似

  • 成员变量就是个变量，声明就有，不声明就没有

2.2 为什么需要方法？

  • 提高代码的复用性
  • 可以将实现的细节封装起来，然后供其他用户来调用即可

比如你要打印个数组，每次打印都要写个for循环，但是如果定义个打印数组的方法printArr，那每次就不用写for循环了，直接调用就行。

2.3 方法的调用

  • 一个方法最多只能有一个返回值，如需返回多个结果，可以返回数组或json字符串
  • 如果方法是void，则方法体中可以没有return语句，或者只写return；
  • 同一个类中的方法，直接调用即可
  • 跨类中的方法，需要通过对象名调用
  • 跨类的方法调用和方法的访问修饰符有关，后面细说

2.4 方法的传参机制

形参和实参的区别：定义方法时的参数叫形参，调用方法时实际传入的参数叫实参，形参名和实参名并没有什么关系。

  • 形参为基本数据类型的话，方法对参数的操作并不影响变量本身
  • 形参为引用数据类型的话，方法对参数的操作就会影响变量本身

3. 递归

递归就是自己调用自己

3.1 递归的特点

  • 执行一个方法时，就创建一个新的受保护的独立空间（栈空间）
  • 方法的局部变量是独立的，不会相互影响
  • 如果方法中使用的是引用类型变量(比如数组，对象)，就会共享该引用类型的数据
  • 递归必须向退出递归的条件逼近，否则就是死循环
  • 当一个方法执行完毕，或者遇到return，就会返回，遵守谁调用，就将结果返回给谁，同时当方法执行完毕或者返回时，该方法也就执行完毕

4. 方法的重载

多个方法同名，但是形参列表不一致，叫方法的重载，解决了起名太麻烦的问题。如果不重载但是重名会报错，因为是方法的重复定义。

5. 可变参数

  • 可变参数的实参可以有0个或多个，它本质是数组，所以也可以传数组
  • 可变参数和普通参数同时存在时，必须保证可变参数在最后，所以一个形参列表中只能有一个可变参数
  • 局部变量必须赋值才能用

6. 变量的作用域

变量分全局变量和局部变量。

  • 全局变量就是属性和成员变量，作用域是整个类，有默认值，可以不赋值直接用
  • 局部变量作用域为当前代码块，无默认值，必须赋值才能用
  • 全局变量和局部变量之间可以重名，访问时遵循就近原则，但是同一个作用域中不能重名
  • 属性生命周期长，随对象创建，随对象销毁
  • 局部变量生命周期短，代码块执行才创建，执行完立刻销毁
  • 全局变量可以在本类中使用，也可以被其他类通过对象调用；局部变量只能在本代码块中使用
  • 全局变量可以加访问修饰符；局部变量不可以

7. 构造方法和构造器

前面我们在创建人类的对象时，是先把一个对象创建好后，再给他的年龄和姓名属性赋值，如果现在我要求，在创建人类的对象时，就直接指定这个对象的年龄和姓名，该怎么做?？这时就可以使用构造器constructor，也叫构造方法。它是类的一种特殊的方法，主要作用是完成对对象的初始化，在创建对象时，系统会自动的调用该类的构造器完成对象的初始化。

  • 构造器没有返回值，也不能写void，可以加修饰符
  • 构造器的名称必须和类名一样
  • 构造器在创建对象时由系统自动调用，来完成对象的初始化，并不是创建对象
  • 一个类可以定义多个不同的构造器，即构造器重载，同方法的重载
  • 如果程序员没有定义构造器，系统会自动给类生成一个默认的无参构造器，可以用javap指令反编译查看
  • 如果程序员定义了构造器，默认的无参构造器就没了，想要就只能手动再定义，这点很重要！！！

2. 访问修饰符

3. 封装

面向对象编程有三大特征：封装、继承和多态。

封装encapsulation，就是把抽象的属性和对数据操作的方法封装在一起，数据被保护在内部，程序的其它部分只有通过被授权的操作[方法]，才能对数据进行操作。

封装的好处：隐藏实现细节，安全可靠。

实现封装的3个步骤：

  • 将属性进行私有化，不让人直接修改属性
  • 提供一个公共的set方法，对属性进行判断并赋值
  • 提供一个公共的get方法，用于获取属性值

4. 继承

当我们编写两个类，这两个类的属性和方法有很多是相同的，能代码复用吗？可以！！！

继承extends可以解决代码复用的问题，让我们的编程更加靠近人类思维。当多个类存在相同的属性(变量)和方法时，可以从这些类中抽象出父类，在父类中定义这些相同的属性和方法，所有的子类不需要重新定义这些属性和方法，只需要通过 extends 来声明继承父类即可。

  • 子类会自动拥有父类定义的属性和方法，但是父类要想调用子类的特有方法，必须向下转型
  • 父类又叫超类基类，子类又叫派生类
  • 提高了代码的复用性、扩展性和可维护性

深入理解继承：

  • 子类继承了所有的属性和方法，非私有的属性和方法可以在子类直接访问，但是私有属性和方法不能在子类直接访问，要通过父类提供公共的方法去访问

  • 子类必须调用父类的构造器，来完成父类的初始化

  • 当创建子类对象时，不管使用子类的哪个构造器，默认情况下总会去调用父类的无参构造器，如果父类没有提供无参构造器，则必须在子类的构造器中用 super 去指定使用父类的哪个构造器完成对父类的初始化工作，否则编译不会通过

  • 如果希望指定去调用父类的某个构造器，则显式的调用一下：super(参数列表)

  • super 在使用时，必须放在构造器第一行(super 只能在构造器中使用)

  • super() 和 this() 都只能放在构造器第一行，因此这两个方法不能共存在一个构造器

  • java 所有类都是 Object 类的子类，Object是所有类的基类

  • 父类构造器的调用不限于直接父类！将一直往上追溯直到 Object 类(顶级父类)

  • 子类最多只能继承一个父类(指直接继承)，即 java 中是单继承机制

  • 思考：如何让 A 类继承 B 类和 C 类？ 【A 继承 B，B 继承 C】

  • 不能滥用继承，子类和父类之间必须满足 is-a 的逻辑关系

  • 注意子类构造器和父类构造器的调用顺序

  • 父子类的查找顺序，先从子类找，没有再去父类找，没有再去父类的父类找

5. super关键字

super 代表父类的引用，用于访问父类的属性、方法、构造器。

  • super调用父类的属性方法时，位置随便，注意不能访问私有的，但是调用父类构造器时，只能放在第一行
  • super不限于直接父类，也可以调用父类的父类中的属性，遵循就近原则，当父类和父类的父类都有某个属性时，调用的是父类中的
  • call()、this.call()、super.call()，这三个的区别是，前俩作用完全相同，根据就近原则找到就返回，super.call()是直接调用父类的方法，自己有也不调用
  • 调用父类构造器可以使分工明确，父类属性由父类初始化，子类属性由子类初始化
  • 当子类和父类中的成员(属性和方法)重名时，为了访问父类的成员，必须通过super，如果没有重名，那么通过super、this或直接访问，效果是一样的

super和this的区别：

区别点 this super
访问属性和方法 访问本类中的属性和方法，如果没有则从父类继续找 直接从父类开始查找属性和方法
调用构造器 调用本类的构造器，必须放在构造器的首行 调用父类的构造器，必须放在构造器的首行
特殊 表示当前类的对象 在子类中访问父类对象

6. 方法的重写

当子类中有一个方法和父类中的某个方法，两者方法名称、形参列表完全一样，那么我们就说子类的这个方法重写了父类的方法。

  • 对于返回值类型，要求子类的返回值类型和父类相同或是父类返回值类型的子类
  • 子类方法不能缩小父类方法的访问权限
  • final、static、private修饰的方法，是不能被重写的？

重写和重载的区别：

7. 多态

方法或对象具有多种形态，是面向对象的第三大特征，多态是建立在封装和继承基础之上的。

7.1 快速入手

先看一个程序：Master类(主人类)中有一个feed(喂食)方法，用来完成主人给动物的喂食。

  • 如果动物只有猫狗，可以重载两个feed方法，但是如果有100种动物呢？
  • 可见传统方式的代码复用性不高，而且不利于代码维护
  • 用多态的写法，100种动物一个方法搞定
  • 当添加新的动物时，直接定义两个类分别继承Animal和Food就可以了

7.2 对象的多态

核心，困难，重点

  • 一个对象的编译类型和运行类型可以不一致
  • 编译类型在定义对象时，就确定了，不能改变；运行类型是可以变化的
  • 编译看左边，运行看右边
  • 多态的前提是，两个类存在继承关系

比如上面的Animal animal = new Cat(“小白”)，编译时是Animal类型的，运行时就变成了Cat类型

7.3 多态的转型

多态的向上转型：父类的引用指向了子类的对象

  • 举例：Animal animal = new Cat()
  • 编译看左边，运行看右边
  • 可以调用父类的所有成员，不能调用子类的特有成员，但是最终运行效果看子类的具体实现
  • 因为在编译阶段，能调用哪些成员,是由编译类型来决定的
  • 注意在大数据里，int类型可以用string来存，但是在java里，integer和string是没有继承关系的包装类，不能强转

多态的向下转型：让向上转型的引用再强转回来

  • 举例：Cat cat2 = (Cat) animal

  • 只能强转父类的引用，不能强转父类的对象，比如animal如果是靠Animal类new出来的，就不能强转

  • 向下转型可以调用子类类型中的所有成员

8. Object类详解

8.1 equals方法

equals和 == 的区别：

  • equals是Object类中的方法，只能判断引用类型是不是同一个对象，即判断地址
  • equals默认判断的是地址是否相等，子类往往重写该方法，用来判断内容是否相等
  • ==既可以判断基本类型又可以判断引用类型，对基本类型判断值，对引用类型判断地址
  • Object类中的equals方法就是用==来判断的，但是包装类和String重写了equals方法，来判断值

8.2 hashCode 方法

  • 提高具有哈希结构的容器的效率，返回值是个整数
  • 两个引用如果指向同一对象，哈希值必相同，指向不同对象，哈希值不一定相同
  • 哈希值主要根据地址号来的，不能完全将哈希值等价于地址
  • 不一定的理解：5%3=2；8%3=2；但5!=8

hashCode的重写，在后面讲字符串的时候说。

8.3 toString 方法

  • 默认返回：全类名+@+哈希值的十六进制，子类往往重写 toString 方法，用于返回对象的属性信息
  • 打印对象或拼接对象时，都会自动调用该对象的 toString 形式，比如sout(cat)
  • Object的toString()源码：return getClass().getName() + “@” + Integer.toHexString(hashCode())

8.4 finalize 方法

  • 当对象被回收时，系统自动调用该对象的 finalize 方法，子类可以重写该方法，做一些释放资源的操作
  • 什么时候被回收：当某个对象没有任何引用时，则 jvm 就认为这个对象是一个垃圾对象，就会使用垃圾回收机制来销毁该对象，在销毁该对象前，会先调用 finalize 方法
  • 垃圾回收机制的调用，是由系统来决定(即有自己的 GC 算法), 也可以通过 System.gc() 主动触发垃圾回收机制
  • 在实际开发中，几乎不会运用 finalize , 所以更多就是为了应付面试

九、面向对象下

1. 类变量

问题引出：有一群小孩在玩堆雪人，不时有新的小孩加入，请问如何知道现在共有多少人在玩？

传统方法：main方法中定义一个变量count，每当一个小孩加入游戏，count++

传统方法的问题是，count是一个独立的变量，访问起来不方便，也没有用到OOP

因此引出类变量，又叫静态变量

  • 类变量，可以通过**类名直接访问(推荐)**，用实例名访问的话不能补全
  • 静态变量是该类的一个变量，会被该类的所有实例共享
  • 静态变量是随着类的加载而创建，所以即使没有创建对象实例也可以访问，随类消亡而销毁
  • 静态变量是该类的所有对象共享，成员变量是每个对象独享，所以成员变量不能通过类名访问

2. 类方法

类方法同类变量

  • 当我们不希望创建对象也能调用方法时，就可以做成静态方法，比如自定义的工具类

  • 静态方法和普通方法都是随着类的加载而加载？，将结构信息储存在方法区

  • 静态方法中不允许使用和对象有关的关键字，比如this和super

  • 静态方法中只能访问静态变量或静态方法，普通方法能访问所有，因为加载的比普通变量早？

  • 但是普通方法和静态方法不是一起加载的？为啥普通方法就可以访问普通变量？

3. main方法

pulic static void main(String[] args){}

  • main方法是虚拟机调用的

  • java虚拟机需要调用类的main()方法，所以该方法的访问权限必须是public

  • java虚拟机在执行main()方法时不必创建对象，所以该方法必须是static

  • 该方法接收String类型的数组参数

  • main方法和静态方法一样，可以直接调用同类的静态方法和静态属性，但是不能直接访问非静态成员，只能通过该类的对象去访问

4. 代码块

类似于没有方法名的方法，没有返回值没有参数只有方法体，用{}包括起来，不用通过对象或类显式调用，在加载类时或创建对象时隐式调用。

  • 代码块的修饰符可以不写，写的话只能是static
  • 代码块后面的；可以不写，但是最好写上
  • 可以看做是对构造器的补充，用来做初始化的作用，如果多个构造器有重复的语句，这个语句就可以写在代码块里，提高代码的复用性
  • 静态代码块随着类的加载而执行，而且只会执行一次，作用是对类进行初始化，而普通代码块，每创建一个对象就会执行一次
  • 静态相关的代码块或属性，在类加载时就执行完毕
  • 普通代码块的优先级也是高于构造器的

类什么时候被加载？

  • 创建对象时
  • 创建子类对象时，父类也会被加载，而且先加载父类
  • 使用类的静态成员时

注意普通代码块只会在创建对象时执行，管他类加不加载

在创建对象时，一个类的调用顺序是：

  • 父类的静态代码块和静态属性初始化，这俩优先级一样，按顺序执行
  • 子类的静态代码块和静态属性初始化
  • 父类的普通代码块和普通属性初始化
  • 父类的构造方法
  • 子类的普通代码块和普通属性初始化
  • 子类的构造方法

所以静态代码块只能直接调用静态成员，普通代码块可以调用任意成员，因为静态代码块加载时，普通成员还没加载。

小练习

5. 单例模式

什么是设计模式

设计模式是在大量的实践中总结优化的代码结构、编程风格以及解决问题的思考方式。它里面有静态方法和属性的经典使用。

什么是单例模式

采取一定的方法保证在整个的软件系统中，对某个类只能存在一个对象实例，并且该类只提供一个取得其对象实例的方法。

单例模式有两种：懒汉式和饿汉式，步骤如下：

  • 构造器私有化，防止直接new
  • 类的内部创建对象
  • 向外暴露一个静态的公共方法用来返回对象

他们两个的区别是：

  • 二者最主要的区别在于创建对象的时机不同：饿汉式是在类加载时就创建了对象实例，而懒汉式是在使用时(调用getInstance)才创建

  • 懒汉式只有当用户调用getInstance时，才返回该类的对象，后面再调用时，会返回第一次调用所创建的对象，从而保证了单例

  • 饿汉式不存在线程安全问题，懒汉式存在线程安全问题（后面学习线程后，会完善一把）

  • 饿汉式可能造成創建了對象，但是沒有使用，造成资源的浪费，饿汉式就不存在这个问题

  • 在我们javaSE标准类中，java.lang.Runtime就是经典的单例模式

6. final关键字

final 可以修饰类、属性、方法和局部变量。使用场景如下：

  • 当不希望类被继承时，可以用final修饰
  • 当不希望父类的某个方法被子类重写时，可以用final关键字修饰
  • 当不希望类的的某个属性的值被修改，可以用final修饰
  • 当不希望某个局部变量被修改，可以使用final修饰

注意事项：

  • final修饰的属性又叫常量，一般用XX_XX命名，比如TAX_RATE，注意常量名全大写

  • final修饰的常量的赋值问题，必须赋值并且以后不能修改，可以给常量赋值的位置有三个：声明时赋值；在构造器中赋值；在代码块中赋值，但是如果是静态常量，就只能在声明时或静态代码块中赋值，不能在构造器中赋值

  • final类不能继承，但是可以实例化对象

  • 如果类不是final类，但是含有final方法，该方法虽然不能被重写，但是可以被继承

  • 如果一个类已经是final类了，就没必要再将方法修饰成final方法了
  • final不能修饰构造器
  • final和static往往搭配使用，效率更高，不会导致类被加载，底层编译器做了优化
  • 包装类都是final类，String也是final类

6. 抽象类

当父类的某些方法，需要声明但是又不确定如何实现时，可以将其声明为抽象方法(abstract)，那么这个类就是抽象类。抽象方法解决的是父类方法的不确定性问题，父类声明不实现，让子类来实现。

  • 所谓抽象方法就是没有实现的方法，即没有方法体，连花括号都没有

  • 当一个类中存在抽象方法时，必须将该类声明为abstract 类，但是抽象类可以没有抽象方法

  • abstract也只能修饰类和方法

  • 如果一个类继承了抽象类，那它必须实现该抽象类的所有抽象方法，除非它自己也声明为抽象类

  • 抽象类的价值更多在于设计，所以在框架和设计模式中使用较多

  • 抽象类无法被实例化，除非new的时候同时实现了抽象方法

  • 抽象方法不能使用private、final和static来修饰，因为这些关键字都是和重写相违背的

7. 模板设计模式

模板设计模式是抽象类的最佳实践，抽象类体现的就是一种模板模式的设计，抽象类作为多个子类的通用模板，子类在抽象类的基础上进行扩展、改造，但子类总体上会保留抽象类的行为方式。

模板设计模式能解决的问题：

  • 当功能内部一部分确定，一部分不确定，可以把不确定的部分暴露出去，让子类去实现

  • 编写一个抽象父类，父类提供了多个子类的通用方法，并把一个或多个方法留给其子类实现，就是一种模板模式

8. 接口

8.1 接口的定义

接口就是给出一些没有实现的方法，封装到一起，到某个类要使用的时候，再根据具体情况实现。接口类似于更加抽象的抽象类，jdk8.0后，允许接口中可以有方法的具体实现。接口体现了程序设计的多态和高内聚低耦合的设计思想。

接口不是很难理解，难的是不知道什么时候使用接口。

  • 接口不能实例化，除非把里面的方法一起实现

  • 接口中所有的方法都是public方法，public可以忽略，接口中的方法可以不用abstract修饰，因为void connect() = public static void connect()

  • 接口的修饰符只能是public和默认，这点和类的修饰符是一样的

  • 普通类实现接口必须把接口的所有方法都实现，抽象类实现接口可以不用实现接口的方法

  • 一个类可以同时实现多个接口，这点和抽象类不一样，继承抽象类只能是单继承

  • 接口中的属性只能是public static final修饰的，在接口中，int num = 1实际上是public static final int num = 1，可以通过接口名直接调用

  • 接口不能继承其它的类，但是可以继承多个别的接口，注意这里，类是单继承，接口是可以多继承的

  • 接口的属性能通过接口名直接调用说明是static修饰，不能改属性值说明是final修饰

8.2 实现接口与继承类的区别

  • 当子类继承了父类，就自动的拥有父类的功能，如果子类需要扩展功能，可以通过实现接口的方式扩展，实现接口可以理解为是对java单继承机制的一种补充

  • 接口和继承解决的问题不同，继承解决的是代码的复用性和可维护性问题，接口的价值在于设计好各种规范方法，让其他类去实现这些方法

  • 接口比继承更加灵活，继承是满足is a的关系，接口只需要满足like a的关系

  • 接口在一定程度上实现了代码的解耦

8.3 接口的多态性

多态参数和多态数组前面都演示过，还有个接口的多态传递，比如self接口继承了parent接口，sub类实现了self接口，相当于sub类也实现了parent接口。

9. 内部类

9.1 内部类的定义

一个类的内部又完整的嵌套了另一个类结构，被嵌套的类(就是里边那个类)称为内部类(inner class)，嵌套其他类的类称为外部类(outer class)是我们类的第五大成员。

内部类最大的特点就是可以直接访问私有属性，并且可以体现类与类之间的包含关系，注意：内部类是学习的难点，同时也是重点，后面看底层源码时，有大量的内部类。

  • 把类定义在局部位置(方法/代码块中)：局部内部类、匿名内部类(没有类名)，局部变量不能加修饰符

  • 把类定义在成员位置：成员内部类、静态内部类(static修饰)，成员变量可以加任意访问修饰符

思考：类的五大成员是哪些？属性、方法、构造器、代码块、内部类

9.2 内部类的特点：

局部位置的内部类：

  • 不能添加访问修饰符，是个类，也是个局部变量，但是可以用final修饰

  • 作用域仅仅是在定义它的方法或代码块中，外部类要想访问内部类的成员，要创建内部类对象，同时也要注意内部类的作用域，外部其他类不能访问内部类

  • 如果外部类和局部内部类的成员重名时，遵循就近原则，如果非要访问外部类的成员，用外部类名.this.成员去访问

  • 外部类.this本质就是外部类的对象，哪个对象调用了method3，外部类.this就是哪个对象

成员位置的内部类：

  • 成员内部类相当于一个全局成员，所以可以加任意访问修饰符，作用域、访问方式也跟全局成员一样

9.3 局部内部类

在方法或代码块中创建个类，这个类可以直接调用外部类的成员包括私有的，这个类的声明和实例化都是在这个方法里。本质是依然是个类，但是作用域是在方法或代码块中。

9.4 匿名内部类

匿名内部类也是定义在方法或代码块中，本质是个类，同时也是一个对象，并且该类没有名字。
