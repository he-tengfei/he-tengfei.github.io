三、JDBC API

JDBC API是一系列的接口，它统一和规范了应用程序和数据库的连接、执行SQL语句、得到返回结果等各类操作，相关类和接口在java.sql和javax.sql包中

  • PreparedStatement
  • DriverManager
  • Statement
  • ResultSet

1.Statement与PreparedStatement

1.1 Statement

  • Statement对象，用于执行静态sql语句并返回其生成的结果的对象
  • 在连接建立后，需要对数据库进行访问，有三种方式
    • Statement，存在SQL注入问题，实际开发不会用
    • PreparedStatement，预处理，真正用的是这个
    • CallableStatement，存储过程
  • Statement对象执行SQL语句，存在SQL注入风险
  • SQL注入是利用某些系统没有对用户输入的数据进行充分的检查，而在用户输入数据中注入非法的SQL语句或命令，来恶意攻击数据库
  • 要防范SQL注入，只要用PreparedStatement(从Statement扩展而来)，取代Statement就可以了

1.3 PreparedStatement

简介：

  • PreparedStatement执行的SQL语句中的参数用占位符？来表示，调用PreparedStatement对象的setxxx()方法来设置这些参数。setxxx()方法有两个参数，第一个参数是从1开始的索引，第二个是参数值
  • 调用executeQuery()，可执行查询，返回ResultSet对象
  • 调用executeUpdate()，可执行更新，包括增删改，返回受影响的行数
  • 对象由connect.prepareStatement()生成

预处理的好处：

  • 不再使用+拼接SQL语句，减少语法错误
  • 有效解决了SQL注入问题
  • prepareStatement会在Java端对sql进行预编译，然后再发给MySQL，MySQL就不用编译了，直接执行，大大减少了编译次数，效率较高

2. ResultSet结果集

  • 表示数据库结果集的数据表，通常通过执行查询数据库的语句生成

  • resultset的指针刚开始是指向第一行之前的，调用一次next()方法，指针下移一行，这时才指向第一行，类似迭代器？

  • next()方法将光标移动到下一行，并且由于在ResultSet对象中没有更多行时返回false，因此可以在while循环中使用循环来遍历结果集

  • 为什么用while循环？返回false，不用就报错！

五、事务和批处理

1. 事务

基本介绍：

  • JDBC程序中，当一个Connection对象创建时，默认情况下是自动提交事务
  • 为了让多个SQL语句作为一个整体执行，需要使用事务
  • 调用Connection的setAutoCommit(false)可以取消自动提交事务
  • 所有SQL语句执行成功后，调用Connection的commit()方法提交事务
  • 如果出现异常，调用Connection的rollback()方法回滚事务

2. 批处理

假如要执行三条sql，默认是把三条sql逐个发送给Connection，可以通过批处理一次性发送过去增加效率。

基本介绍

  • 当需要成批插入或者更新记录时，可以采用Java的批处理机制，这一机制允许多条语句一次性提交给数据库批量处理，通常情况下比单独提交处理更有效率
  • JDBC的批量处理语句包括下面方法：
    • addBatch()，添加需要批量处理的SQL语句或参数
    • executeBatch()，执行批量处理语句
    • clearBatch()，清空批处理的语句
  • JDBC连接MySQL时，如果要使用批处理功能，必须在URL中添加参数?rewriteBatchedStatements=true
  • 批处理往往和PreparedStatement一起搭配使用，可以既减少编译次数，又减少运行次数，效率大大提高

六、连接池

1. 为什么要使用连接池？

  • 传统的JDBC数据库连接使用DriverManager来获取，每次向数据库建立连接的时候都要将Connection加载到内存中，再验证IP地址，用户名和密码(耗时0.05s~1s)。频繁的进行数据库连接操作将占用很多的系统资源，容易造成服务器崩溃
  • 每一次数据库连接，使用完都得断开，如果程序出现异常而未能关闭，将导致数据库内存泄漏，最终将导致重启数据库
  • 传统获取连接的方式，不能控制创建的连接数量，如连接过多，也可能导致内存泄漏，MySQL崩溃
  • 解决传统开发中的数据库连接问题，可以采用数据库连接池技术(connection pool)

问题演示：

2. 数据库连接池介绍

  • 预先在缓冲池中放入一定数量的连接，当需要建立数据库连接时，只需从缓冲池中取出一个，使用完毕之后再放回去
  • 预先放在连接池中的连接，如果不手动关闭是不会断开的，会一直保持连接
  • 数据库连接池负责分配、管理和释放数据库连接，它允许应用程序重复使用一个现有的数据库连接，而不是重新建立一个
  • 当应用程序向连接池请求的连接数超过最大连接数量时，这些请求将被加入到等待队列中

3. 常用连接池种类

JDBC的数据库连接池使用javax.sql.DataSource来表示，DataSource是个接口，由第三方实现

  • C3P0数据库连接池，速度相对较慢，很稳定，这里的慢是相对的，实际还是很快，老牌的连接池了
  • DBCP数据库连接池，速度相对C3P0较快，但不稳定
  • Proxool数据库连接池，有监控连接池状态的功能，稳定性稍差
  • BoneCP数据库连接池，速度快
  • **Druid(德鲁伊)**是阿里提供的数据库连接池，集上述四种优点于一身，推荐使用

七、Apache–DBUtils

1. 为什么需要DBUtils？

现有JDBCUtils存在的问题是，关闭connection后，resultSet结果集无法再使用，我查询完数据肯定就把连接给关了，但是我想对我查到的结果进行各种再分析，这就无法实现了。说白了就是解决resultset结果集无法复用的问题

土办法封装的思路：

假设一张表，里面有3个字段：id、name、sex，建个只有这三个属性的类Actor，把表里的一行数据传给类Actor的一个对象，再把类Actor的对象放到ArrayList集合中，这样表里有几行数据，就会有几个Actor对象，都给它放到集合里，这个只有属性的类就叫JavaBean或者PoJo类。
