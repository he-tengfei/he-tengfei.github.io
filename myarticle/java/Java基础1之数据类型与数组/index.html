一、Java的运行机制

1. Java概述

什么是程序

程序：计算机执行某些操作或解决某个问题而编写的一系列有序指令的集合。

  • Java SE：Java Standard Edition，标准版，面向桌面级应用
  • Java EE：Java Enterprise Edition，企业版，面向企业环境，主要针对web应用程序开发，核心技术servlet、jsp等
  • Java ME：Java Micro Edition，小型版

Java重要特点

  • Java语言是面向对象的，oop

  • Java 语言是健壮的。Java 的强类型机制、异常处理、垃圾的自动收集等，是 Java 程序健壮性的重要保证

  • Java 语言是跨平台性的。[即: 一个编译好的.class文件可以在多个系统下运行比如Windows系统或者Linux系统，这种特性称为跨平台]

  • Java 语言是解释型的

    • 解释性语言：PHP，java；编译性语言: c / c++
    • 区别是：解释性语言，编译后的代码，不能直接被机器执行，需要解释器来执行；编译性语言，编译后的代码，可以直接被机器执行，比如c /c++

2. Java 语言的特点：跨平台性

有了JVM，同一个java程序可以在三个不同的操作系统中执行。

3. JVM

Java的核心机制：Java虚拟机JVM，java virtual machine

  • JVM 是一个虚拟的计算机，具有指令集并使用不同的存储区域。负责执行指令，管理数据、内存、寄存器，包含在JDK 中
  • 对于不同的平台，有不同的虚拟机
  • Java 虚拟机机制屏蔽了底层运行平台的差别，实现了“一次编译，到处运行”

4. 什么是JDK，JRE

4.1 JDK基本介绍

  • JDK的全称 Java Development Kit，Java开发工具包
  • JDK = JRE + java的开发工具[java,javac,javadoc,javap等]
  • JDK是提供给Java开发人员使用的，其中包含了java的开发工具，也包括了JRE。所以安装了JDK，就不用再单独安装JRE了

4.2 JRE基本介绍

  • JRE全称 Java Runtime Environment，Java运行环境
  • JRE = JVM + Java的核心类库[类]
  • 包括Java虚拟机JVM和Java程序所需的核心类库等，如果想要运行一个开发好的Java程序，计算机中只需要安装JRE即可

4.3 JDK、JRE和JVM的包含关系

  • JDK=JRE+开发工具集（例如Javac,java编译工具等)
  • JRE=JVM+JavaSE标准类库（java核心类库）
  • 如果只想运行开发好的.class文件只需要JRE

4.4 JDK的安装

见软件安装篇，注意安装路径下不要有空格或特殊符号或中文！！！

三、变量与数据类型

1. 变量

不论哪种高级程序语言，变量都是程序的基本组成单位，它有三个基本要素：数据类型、变量名、变量值。

  • 变量表示内存中的一个存储区域[不同的变量，类型不同，占用的空间大小不同，比如：int4个字节，double 就是8个字节，先有基本印象，后面说字节]
  • 该区域有自己的名称[变量名]和类型[数据类型]
  • 变量必须先声明，后使用，即有顺序
  • 该区域的数据/值可以在同一类型范围内不断变化
  • 变量在同一个作用域内不能重名
  • 变量=变量名+值+数据类型，这一点请大家注意，变量三要素

2. 标识符的命名

2.1 标识符概念

  • Java 对各种变量、方法和类等命名时使用的字符序列称为标识符
  • 简单来说，凡是自己可以起名字的地方都叫标识符

2.2 命名基本准则

  • 由26个英文字母大小写，0-9，_或$组成
  • 数字不可以开头
  • 不可以使用关键字和保留字，但能包含关键字和保留字
  • Java中严格区分大小写，长度无限制
  • 标识符不能包含空格
  • 最好采用驼峰命名法，比如int totalNum = 10;

2.3 命名行业规范

  • 包名：所有字母都小写，比如 com.study
  • 类名、接口名：所有单词的首字母都大写，比如HelloWorld
  • 变量名、方法名：第一个单词首字母小写，第二个单词开始每个单词首字母大写，比如toString()
  • 常量名：所有字母都大写，多单词时每个单词用下划线连接，比如，TAX_RATE

3. 关键字和保留字

3.1 关键字

  • 被 Java 语言赋予了特殊含义，用做专门用途的单词
  • 关键字中所有字母都为小写

3.2 保留字

现有 Java 版本尚未使用，但以后版本可能会作为关键字使用。自己命名标识符时要避免使用这些保留字：by、value、cast、future、generic、inner、operator、outer、rest、var、goto、const

4. 数据类型

每一种数据都定义了明确的数据类型，在内存中分配了不同大小的内存空间(字节)。

java数据类型分两大类，基本数据类型8个和引用数据类型3个，必背！！！

  • 基本数据类型：byte[1], short[2], int[4], long[8], float[4], double[8], char[2], boolean[1]
  • 引用数据类型：类class, 接口interface, 数组[]

数据类型的使用细节：

  • java整型常量默认为int，声明long时要加L，比如long n = 1L
  • 1byte = 8 bit
  • java的浮点型常量默认为double，精度比较高，禁用float
  • 浮点型的使用陷阱：double num = 8.1/3; System.out.println(num); //2.6999999999999997
  • double num = 8; sout(num); //8.0
  • 字符类型是 char，char 是两个字节(可以存放汉字)，多个字符用字符串 String
  • char必须用单引号，它的本质其实是个整数，可以进行运算，输出unicode码对应的字符

5. 常见编码简介

字符和码值的对应关系是通过字符编码表决定的（是规定好），下面是常见字符编码表

  • ASCIl 编码表，上个世纪60年代，美国指定了一套字符编码，使用一个字节对英文字符和二进制位之间的关系做了统一规定。这被称为ASCII码。ASCII码一共规定了128个字符的编码，只占用了一个字节的后面7位，最前面的1位统一规定为0。一个字节可以表示256个字符，ASCII码只用了128个字符。它的缺点是不能表示所有字符，比如某些汉字
  • Unicode 编码表，将世界上所有的符号都纳入其中。每一个符号都给予一个独一无二的编码，使用Unicode 没有乱码的问题。但是一个英文字母和一个汉字都占用2个字节，浪费空间，Unicode码兼容ASCll码
  • UTF-8编码表，是在互联网上使用最广的一种Unicode的实现方式，采用变长的编码方式，根据不同的符号而变化字节长度，字母占1个字节，汉字占3个字节
  • 字节与位：一个字节是8位，它可以表示2^8 = 256个字符，一个中文占两个(ASCII)或3个字节(UTF-8)，一个字母占1个字节
  • 字符计算机中使用的字母、数字、字和符号，比如’A’、’B’、’$’、’&’等，所以上面说的1个汉字占3个字节，就是在说字符与字节的关系

6. 数据类型转换

6.1 自动类型转换

当java程序运算时，精度小的类型会自动转换为精度大的数据类型，叫做自动类型转换。

  • (char -> int 或者 byte -> short -> int) -> long -> float -> double
  • 比如double num = 80时，80会做自动类型转换为80.0
  • byte、short、char之间不能自动转换，但是可以运算，运算时转为int类型

6.2 强制类型转换

将容量大的数据类型转为容量小的数据类型时要进行强转，否则会报错，强转的话可能造成精度丢失或溢出。

  • int a = (int)1.9

6.3 基本数据类型和String类型的转换

  • 基本数据类型转String，用 + “” 即可，比如 String str = 100 + “”

  • String转基本数据类型，调用包装类即可，比如Integer.parseInt(“123”)

  • 要得到字符串的某个字符，可以用str.charAt(0)得到

7. 二进制

对于整数，有四种表示方式：

  • 二进制：0,1 ，满 2 进 1，以 0b 或 0B 开头
  • 十进制：0-9 ，满 10 进 1
  • 八进制：0-7 ，满 8 进 1. 以数字 0 开头表示

7.1 二进制和十进制的转换

7.2 二进制在运算中的说明

现代的电子计算机技术全部采用的是二进制，因为它只使用0、1两个数字符号，非常简单方便，易于用电子方式实现。计算机内部处理的信息，都是采用二进制数来表示的。二进制（Binary）数用0和1两个数字及其组合来表示任何数。进位规则是“逢2进1”，数字1在不同的位上代表不同的值，按从右至左的次序，这个值以二倍递增。

7.3 原码、反码、补码

  • 原码：原码是一种计算机中对数字的二进制定点表示方法，一般进制的最高位是符号位，1代表负号，0代表正号

  • 反码：正数的反码跟原码一样；负数的反码是在原码的基础上，符号位不变，其他位0变1，1变0

  • 补码：正数的补码与原码一样；负数的补码在相应的反码上+1；

  • 正数三码合一，由于二进制的第一位是符号位，所以java中的数严格来说都是有符号的

  • 计算机运算的时候，都是以补码的方式来运算的，但是我们看运算结果的时候，要看他的原码

8. 位运算

8.1 位运算规则

java 中有 7 个位运算符，分别是按位与&、按位或|、按位异或^，按位取反~，算术右移 >>，算术左移 <<，无符号右移>>>。

  • 按位与&：两位全为1，结果为1，否则为0
  • 按位或|：两位有一个为1，结果为1，否则为0
  • 按位异或^：两位一个为0，一个为1，结果为1，否则为0
  • 按位取反~：0->1，1->0
  • 算术右移 >>：低位溢出，符号位不变，并用符号位补溢出的高位
  • 算术左移 <<: 符号位不变，低位补 0
  • 无符号右移 >>>：低位溢出，高位补 0
  • 注意没有<<<这个符号！！！

6. 排序

排序分两种，内部排序和外部排序。

内部排序：指将需要处理的所有数据都加载到内部存储器中进行排序，包括(交换式排序法、选择式排序法和插入式排序法)

外部排序：数据量过大，无法全部加载到内存中，需要借助外部存储进行排序，包括(合并排序法和直接合并排序法)

7. 查找

查找也是两种，顺序查找和二分查找。顺序查找就是遍历，二分查找见算法篇。

8. 二维数组

把几个一维数组组成个数组，就是二维数组，也就是说，二维数组的每个元素，都是一维数组。

动态初始化：int a[][]=new int[2][3]，2表示该二维数组有两个元素，后面那个3可以给也可以不给，当二维数组每个元素的长度确定时，就可以给，如果不确定，就可以不给。注意二维数组也有默认值。

二维数组的空间是什么时候开辟的呢？比如int a[][]=new int[2][]会开辟空间吗？每个元素的长度不确定该开辟多少呢？
