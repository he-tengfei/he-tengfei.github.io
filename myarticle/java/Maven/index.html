一、Maven简介

Maven，自动化的构建工具

1. 软件开发中的阶段

  • 需求分析： 分析项目具体完成的功能，有什么要求， 具体怎么实现。(需求分析师来做)

  • 设计阶段：根据分析的结果， 设计项目的使用什么技术， 解决难点。

  • 开发阶段：编码实现功能。 编译代码。自我测试

  • 测试阶段：专业的测试人员，测整个项目的功能十分符合设计要求。出一个测试报告。

  • 项目的打包，发布阶段： 给用户安装项目

后面三个阶段由我们开发直接参与，其中有些工作是重复多次的，可以交给maven来做

2. Maven能做什么

  • 项目的自动构建，帮助开发人员做项目代码的清理，编译，测试，报告，打包，安装，部署等工作

  • 管理依赖（管理项目中使用的各种jar包）

依赖：项目中需要使用的其他资源，常见的是jar，比如项目要使用mysql驱动，我们就说项目依赖mysql驱动。

那么不使用maven怎么管理依赖呢？

  • 需要从网络中单独下载某个jar

  • 需要选择正确版本

  • 手工处理jar包之间的依赖，比如a.jar里面要使用b.jar的类

3 什么是maven

maven是apache基金会的开源项目，使用java语法开发。 Maven 这个单词的本意是：专家，内行。它是项目的自动化构建工具，也可以管理项目的依赖。

maven中的9个核心概念：

  • POM
  • 约定的目录结构
  • 坐标
  • 依赖管理
  • 仓库管理
  • 生命周期
  • 插件和目标
  • 继承
  • 聚合

二、Maven的核心概念

1. 约定的目录结构

maven的使用方式：

  • maven可以独立使用： 创建项目，编译代码，测试程序，打包，部署等等

  • maven和idea一起使用：通过idea借助maven，实现编码，测试，打包等等

2. POM文件

POM：Project Object Model 项目对象模型，maven把项目当做模型处理。操作这个模型就是操作项目。

maven通过pom.xml文件实现项目的构建和依赖的管理。

3. 坐标gav

坐标组成是groupid, artifiactId, version，坐标概念来自数学。

坐标作用：用来确定资源，是资源的唯一标识。在maven中，每个资源都是坐标。坐标值是唯一的，简称叫gav。

项目使用gav：

  • 每个maven项目，都需要有一个自己的gav

  • 管理依赖，用坐标表示你需要使用哪个依赖，也需要使用gav作为标识

4. 依赖dependency

依赖：项目中要使用的其他资源（jar包）。

用Maven管理依赖，需要在pom.xml文件中，使用dependencies、dependency和gav完成依赖的说明

6. 仓库

仓库是存东西的，maven的仓库存放的是：

  • maven工具自己的jar包
  • 第三方的其他jar，比如项目中要使用mysql驱动
  • 自己写的程序，可以打包为jar，存放到仓库，比如自定义函数？

Maven中的仓库分两种，本地仓库和远程仓库。

6.1 本地仓库

  • 本地仓库位于你自己的计算机，它是磁盘中的某个目录

  • 默认路径是当前用户根目录/.m2/repository，比如C:\Users\NING MEI\.m2\repository

修改本地仓库的位置：

  • 创建一个目录，作为仓库使用，目录不要有中文和空格

  • 修改maven工具的配置文件setting.xml，指向G:\1workspace\Maven-3.8.2\repository这个目录

同时注意配置阿里云镜像源

6.2 远程仓库

远程仓库：需要通过联网访问的

  • 中央仓库：一个ftp服务器，存放了所有的资源
  • 中央仓库的镜像：就是中央仓库的拷贝，在各大主要城市都有镜像
  • 私服：在局域网中使用的，私服就是自己的仓库服务器，在公司内部使用的

maven仓库的工作方式：maven自动使用仓库，当项目启动后，执行了maven的命令，maven首先访问的是本地仓库，从仓库中获取所需的jar包，如果本地仓库没有，需要访问私服或者中央仓库或者镜像。

7. maven的生命周期，插件和命令

  • maven的生命周期：项目构建的各个阶段。包括清理，编译，测试，报告，打包，安装，部署
  • 插件：要完成构建项目的各个阶段，要使用maven的命令，执行命令的功能是通过插件完成的。插件就是jar包，一些类
  • 命令：执行maven功能是由命令发出的，比如mvn compile

7.1 单元测试（junit）

  • 单元：在java中指的是方法，一个方法就是一个单元，方法是测试的最小单位

  • 作用：使用junit去测试方法是否完成了要求，开发人员自测

maven默认处理配置文件的两条规则

  • maven会把src/main/resources目录中的所有文件，拷贝到target/classes目录下

  • maven只处理src/main/java目录中的.java文件，把这些java文件编译为class，拷贝到target/classes目录中，不处理其他文件

六、Maven管理多模块应用

1. 使用场景

什么场景下会用到Maven管理多模块呢？

有这么一个场景，一个需求由4个模块组成，最终目的是通过application模块实现整体需求，但是它依赖于业务模块bussinessModel，而业务模块又依赖于基础服务模块commonModel，同时业务模块和基础服务模块又都依赖于第三方模块，这种情况下就很适合用Maven来管理这几个模块。比如多个模块都用到了mysql依赖，那这个mysql依赖的版本号是不是要一致呢？

Maven 管理多模块应用的实现是互联网项目中多使用分布式开发，那么每个独立的服务都会使用独立的项目进行维护，那么这样就需要使用多模块应用管理，来实现项目的高度统一。

  • commonModel：提供公共的基础服务，比如工具类、常量类等等；
  • bussinessModel：业务模块，是系统真正要实现的业务，依赖于common模块，比如订单管理、财务统计、会员管理等；
  • application：可发布的web应用，由各个bussinessModel组成，最终满足项目整体需求；
  • 第三方模块：包括各类框架，Spring、MyBatis、日志等。整个应用都是依赖它们完成开发的；

那么如何使用Maven来管理以上的结构呢？有两种实现方式。

复习几个知识点：

  • 正常情况下Maven工程根目录下有三个东西，src、target、pom，其中clean清除的就是target，src也可以没有，只有pom是必须的
  • packaging标签是指定打包的方式，没有的话默认为jar
  • pom文件是项目对象模型(project object module)，该文件是可以被子工程继承的
  • Maven多模块管理就是让它子模块的pom文件来继承父工程的pom文件

Maven父工程必须遵循以下两点：

  • packaging标签的文本内容必须设置为pom

重启后也可能会遇到模组加载不全的情况，等一会儿，让它缓存一下，然后刷新Maven即可。
