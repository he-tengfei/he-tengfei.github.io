Linux篇

一、内容介绍

Linux的应用领域：

  • 个人桌面领域

    • 基本不用

  • 服务器领域

    • linux在服务器领域的应用是最强的。linux免费、稳定、高效等特点在这里得到了很好的体现，尤其在一些高端领域尤为广泛(c/c++/php/java/python/go)

  • 嵌入式领域

    • linux运行稳定、对网络的良好支持性,低成本，且可以根据需要进行软件裁剪，内核最小可以达到几百KB等特点，使其近些年来在嵌入式领域的应用得到非常大的提高
    • 主要应用：机顶盒、数字电视、网络电话、程控交换机、手机、PDA、智能家居、智能硬件等都是其应用领域。以后在物联网中应用会更加广泛

二、Linux概述

1. 概述

  • 不要纠结linux怎么读,不下10种

  • linux是一个开源、免费的操作系统，其稳定性、安全性、处理多并发已经得到业界的认可，目前很多企业级的项目(c/C++ /php/python/java/go)都会部署到Linux/unix系统上

  • 常见的操作系统(windows、ios、Android、MacOS,Linux, Unix)

  • Linux的吉祥物是个小企鹅

  • Linux和Git的创作者是同个人，世界著名黑客Linus，最初的0.01版不到10000行，现在已经超过了700万行

  • Linux主要的发行版：Ubuntu(乌班图，开发python常用这个)、RedHat(小红帽)、CentOS、Debain[蝶变]、Fedora、SuSE、OpenSUSE[示意图]

  • Linux和发行版的关系：Linux是个内核，别人在这个内核上进行的包装就叫发行版

2. Linux和Unix的关系

贝尔实验室的两个人用C语言写了Unix，各大公司对其进行包装和垄断，并且只能运行到大型服务器上。这时有个人认为，在自由的时代，用户应该免费享有对软件源代码阅读、修改的权利。软件公司可以靠提供服务和训练获得盈利。于是他发起了GUN计划，Linus参与其中，对Minix进行了包装，使其能运行到个人计算机上，Linux由此诞生。

四、网络连接的三种方式

张三李四王五是能互相通讯的，因为处在同一个网段192.168.0下，张三采用桥接模式，张三建了个虚拟机，如果用桥接的话，网段必须跟张三的一样，这台虚拟机能跟同网段的李四王五通讯，但是这样就会占用公共网段，192.168.0下面一共只有255个ip。有这么个情况，一共教室200人，每个人要安装个虚拟机，显然就需要400个不同的ip了，但是一共才255个，不满足。而且要把192.168.0.1和.255排除掉，一般.1做网关用，.255做广播用，所以这种方式容易造成ip冲突。另外张三的虚拟机，其实是不想让它跟李四王五互相通讯的，自己练习用的虚拟机，别人能访问也不安全。

王五用nat模式装了个虚拟系统，nat模式会在主机上生成对应的虚拟网卡，范围比如说是192.168.100.99，虚拟机的ip可以设为192.168.100.88，此时99和88会形成一个局域网，可以互通，而且88可以通过主机40的代理访问张三和李四，但是张三李四不能反过来访问88，这样王五的虚拟机能跟外网互通，且不占用公用网段，不会造成ip冲突

六、Linux的目录结构

1. 基本介绍

linux的文件系统是采用级层式的树状目录结构，在此结构中的最上层是根自录“/”，然后在此目录下再创建其他的目录，zookeeper也是这种目录结构。

深刻理解linux树状文件目录是非常重要的，这里我给大家说明一下

记住一句经典的话：在 Linux 的世界里，一切皆文件！！！

2. 目录结构

  • /bin (usr/bin，/usr/local/bin)，是Binary的缩写,这个目录存放着最经常使用的命令
  • /sbin (usr/sbin，/usr/local/sbin)，s就是Super User的意思，这里存放的是系统管理员使用的系统管理程序
  • /home，存放普通用户的主目录，在Linux中每个用户都有一个自己的目录，一般该目录名是以用户的账号命名
  • /root，该目录为系统管理员，也称作超级权限者的用户主目录
  • /opt，这是给主机额外安装软件所存放的目录，比如安装ORACLE数据库就可放到该目录下，默认为空
  • /var，这个目录中存放着在不断扩充着的东西，习惯将经常被修改的目录放在这个目录下，包括各种日志文件，或者mysql的日志和数据文件
  • /etc，所有的系统管理所需要的配置文件和子目录，比如配置环境变量，这个目录操作的很频繁
  • /usr，这是一个非常重要的目录，用户的很多应用程序和文件都放在这个目录下，类似与windows下的program files目录
  • /usr/local，这是另一个给主机额外安装软件所安装的目录，一般是通过编译源码方式安装的程序
  • /tmp，这个目录是用来存放一些临时文件的，可以随便删，不要把重要文件放里面，但是有些框架会默认把一些日志文件放这里
  • /lib，系统开机所需要最基本的动态连接共享库，其作用类似于Windows里的DLL文件，几乎所有的应用程序都需要用到这些共享库
  • /lost+found，这个目录一般情况下是空的，当系统非法关机后，这里就存放了一些文件
  • /boot，存放的是启动Linux时使用的一些核心文件，包括一些连接文件以及镜像文件
  • /proc，这个目录绝对不能动，是一个虚拟的目录，它是系统内存的映射，访问这个目录来获取系统信息
  • /srv，这个目录绝对不能动，service缩写，该目录存放一些服务启动之后需要提取的数据
  • /sys，这个目录绝对不能动，这是linux2.6内核的一个很大的变化。该目录下安装了2.6内核中新出现的一个文件系统
  • /dev，类似于windows 的设备管理器，把所有的硬件用文件的形式存储
  • /media，linux系统会自动识别一些设备，例如U盘、光驱等等，当识别后，linux 会把识别的设备挂载到这个目录下
  • /mnt，系统提供该目录是为了让用户临时挂载别的文件系统的，我们可以将外部的存储挂载在/mnt/上，然后进入该目录就可以查看里面的内容了
  • /selinux [security-enhanced linux]，SELinux是一种安全子系统，它能控制程序只能访问特定文件，有三种工作模式，可以自行设置

Linux系统会内置vi文本编辑器，Vim 具有程序编辑的能力，可以看做是vi的增强版本，可以主动的以字体颜色辨别语法的正确性，方便程序设计、代码补全、编译及错误跳转等方便编程的功能特别丰富，在程序员中被广泛使用。

八、服务

1. 进程运行级别

7种运行级别说明：

  • 0：关机
  • 1：单用户【找回丢失密码】，root权限，用于系统维护，禁止远程登录
  • 2：多用户状态，没有网络服务
  • 3：完全多用户状态，有网络服务，工作中一般是这个，最小化安装
  • 4：系统未使用，保留给用户
  • 5：图形界面
  • 6：系统重启

常用运行级别是3和5，也可以指定默认运行级别，通过init来切换不同的运行级别，具体切换方法不用记，用的时候百度

CentOS7的运行级别简化为: 

multi-user.target 等价于原运行级别3（多用户有网，无图形界面）
graphical.target  等价于原运行级别5（多用户有网，有图形界面）

Linux系统中为了提高磁盘的读写效率，对磁盘采取了 “预读迟写”操作方式。当用户保存文件时，Linux核心并不一定立即将保存数据写入物理磁盘中，而是将数据保存在缓冲区中，等缓冲区满时再写入磁盘，这种方式可以极大的提高磁盘写入数据的效率。但是，也带来了安全隐患，如果数据还未写入磁盘时，系统掉电或者其他严重问题出现，则将导致数据丢失。使用sync指令可以立即将缓冲区的数据写入磁盘。

十、组管理和权限管理

Linux系统是一种典型的多用户系统，不同的用户处于不同的地位，拥有不同的权限。为了保护系统的安全性，Linux系统对不同的用户访问同一文件（包括目录文件）的权限做了不同的规定。在Linux中我们可以使用ll或者ls -l命令来显示一个文件的属性以及文件所属的用户和组。

6个基础命令：groupadd创建组，useradd创建用户，usermod修改用户所在组，chmod变更权限，chown变更文件所有者，chgrp修改文件所在组

权限的优先级也是从前到后的，比如所有者权限不可读，所在组权限可读，那这个用户还是读不了文件

所有者所在组其他组之间的权限互不影响，都可单独设置

比如我有这个文件的操作权限，但是我没有这个文件所在目录的权限，那我可能连目录都进不去，更别说操作文件了

会有这么个情况，比如妖怪组能编辑一个文件，神仙组不能，但是把神仙组的唐三藏添加到妖怪组他还是不能编辑这个文件，重新连接下客户端就可以了，这是什么原理？？？

3. 组管理

为什么要有用户组这个东西呢？方便控制用户权限。

在linux中，每个用户必须属于一个组，不能独立于组外。文件同样，每个文件都有所有者、所在组、其它组的概念，所在组和其他组对文件的权限是不一样的，并且所有者和所在组可以单独修改。用户组的增加、删除和修改实际上就是对/etc/group文件的更新。

  • 所有者：谁创建了文件谁就是所有者
  • 所在组：对于用户来说，如果新建用户没有指定所在组，系统会创建个同名的用户组并把该用户放进去，每个用户必须在某个组里面；对于文件来说，该文件创建者所在的组就是该文件的所在组
  • 除文件所有者和文件所在组的用户外，别的组都是其它组

  • 第0位确定文件类型（d，-，l，c，b），d是目录，相当于windows的文件夹；-是普通文件；l是链接，相当于windows的快捷方式；c是字符设备文件，鼠标，键盘；b是块设备，比如硬盘
  • 第1-3位确定所有者拥有该文件的权限，User
  • 第4-6位确定所属组拥有该文件的权限，Group
  • 第7-9位确定其他组拥有该文件的权限，Other

5.2 权限的表示和作用域

  • r：表示可以对文件用ll命令，将目录的内容显示
  • w：表示可以在该目录，删除或者创建文件
  • x：表示可以进入到该目录，比如cd，或者执行该文件

rwx权限可以用字母表示，也可用数字表示为：r=4，w=2，x=1，因此rwx=4+2+1=7，数字可以进行组合，权限可作用于文件和目录

rwx作用于文件：

r代表可读read；w代表可写write；x代表可执行execute，一般显示绿色

注意write可以修改，但是不代表可以删除该文件，删除一个文件的前提条件是对该文件所在的目录有写权限

rwx作用于目录：

r代表可读read；w代表可写，可修改，对目录内创建+删除+重命名目录；x代表可进入这个目录

2. at定时任务

关于at定时任务，只需要知道它是指定一次性定时任务的就可以了，感觉它的时间指定比较不严谨，现在只需要了解下面这么多即可，用的时候再百度或者翻韩老师笔记吧！

  • at命令是一次性定时计划任务，at的守护进程atd会以后台模式运行，检查作业队列来运行
  • 默认情况下，atd守护进程每60秒检查作业队列，有作业时，会检查作业运行时间，如果时间与当前时间匹配，则运行此作业
  • at命令是一次性定时计划任务，执行完一个任务后不再执行此任务了
  • 在使用at命令的时候，一定要保证atd进程的启动，可以使用相关指令来查看
  • ps-ef I grep atd，这个命令可以检测atd是否在运行，ps -ef检测所有正在运行的进程

十二、磁盘的分区和挂载

磁盘能分区，Linux的目录和磁盘的分区是相对应的，指定哪个分区和哪个目录相对应的操作叫挂载，目录就相当于磁盘分区的快捷方式。就比如下图，sda硬盘有3个分区，分别挂载到/boot、swap和根目录下

1. Linux的分区

1.1 分区简介

Linux无论有几个分区，分给哪一目录使用，它归根结底就只有一个根目录，一个独立且唯一的文件结构，Linux中每个分区都是用来组成整个文件系统的一部分。

Linux采用了一种叫“载入”的处理方法，它的整个文件系统中包含了一整套的文件和目录，且将一个分区和一个目录联系起来，把目录当做分区的快捷方式，访问目录即访问分区。

1.2 Linux硬盘说明

Linux硬盘分IDE硬盘和SCSI硬盘，目前基本上是SCSI硬盘

对于IDE硬盘，驱动器标识符为“hdx”，其中“hd”表明分区所在设备的类型，这里是指IDE硬盘了。“x”为盘号（a为基本盘，b为基本从属盘，c为辅助主盘，d为辅助从属盘）“”代表分区，前四个分区用数字1到4表示，它们是主分区或扩展分区，从5开始就是逻辑分区。例，hda3表示为第一个IDE硬盘上的第三个主分区或扩展分区，hdb2表示为第二个IDE硬盘上的第二个主分区或扩展分区。

对于SCSI硬盘则标识为“sdx～”，SCSI硬盘是用“sd”来表示分区所在设备的类型的，其余则和IDE硬盘的表示方法一样

4. 主机名解析过程分析

Hosts是什么

一个文本文件，用来记录IP和Hostmame（主机名）的映射关系。

DNS是什么

DNS，就是Domain Name System的缩写，翻译过来就是域名系统，是互联网上作为域名和IP地址相互映射的一个分布式数据库。

应用实例：用户在浏览器输入了www.baidu.com会发生什么？

  • 浏览器先检查浏览器缓存中有没有该域名解析IP地址，有就先调用这个IP完成解析；如果没有，就检查DNS解析器缓存，如果有直接返回IP完成解析。这两个缓存，可以理解为本地解析器缓存
  • 一般来说，当电脑第一次成功访问某一网站后，在一定时间内，浏览器或操作系统会缓存他的IP地址（DNS解析记录）如在cmd窗口中输入ipconfig /displaydns可以查看DNS 域名解析缓存，输入ipconfig /flushdns可以手动清理dns缓存
  • 如果本地解析器缓存没有找到对应映射，检查系统中hosts文件中有没有配置对应的域名IP映射，如果有，则完成解析并返回
  • 如果本地DNS解析器缓存和hosts文件中均没有找到对应的IP，则到域名服务DNS进行解析域

十四、进程管理

1. 基本介绍

  • 每个执行的程序或命令都称为一个进程，每一个进程都分配一个pid进程号，都有自己的地址空间，并占用一定的系统资源
  • 每个进程都可能以两种方式存在的。前台与后台，所谓前台进程就是用户目前的屏幕上可以进行操作的。后台进程则是实际在操作，但由于屏幕上无法看到的进程，通常使用后台方式执行
  • 一般系统的服务都是以后台进程的方式存在，而且都会常驻在系统中。直到关机才才结束

2. 查看进程

如果想查看进程的CPU占用率和内存占用率，可以使用ps -aux;
如果想查看进程的父进程ID可以使用ps -ef;

ps -aux：查看系统中所有进程

用来查看目前系统中，有哪些正在执行，以及它们执行的状况，可以不加任何参数。它一共展示4个字段：PID进程识别号；TTY终端机号；TIME此进程消耗CPU时间；CMD正在执行的命令

  • USER：用户名称
  • PID：进程号
  • %CPU：进程占用CPU的百分比
  • %MEM：进程占用物理内存的百分比
  • VSZ：进程占用的虚拟内存大小（单位：KB）
  • RSS：进程占用的物理内存大小（单位：KB）
  • TTY：终端名称，缩写
  • STAT：进程状态，其中S-睡眠，s-表示该进程是会话的先导进程，N-表示进程拥有比普通优先级更低的优先级，R-
    正在运行，D-短期等待，Z-僵死进程，T-被跟踪或者被停止等等
  • START：进程的启动时间
  • TIME：进程使用CPU的总时间
  • COMMAND：启动进程所用的命令和参数，如果过长会被截断显示

ps -ef：查看父子进程之间的关系

以全格式显示当前所有的进程，查看进程的父进程，-e显示所有进程，-f全格式

比如查看sshd的父进程信息：ps-ef l grep sshd

是BSD风格

UID：用户ID

PID：进程ID

PPID：父进程ID

C：CPU用于计算执行优先级的因子。数值越大，表明进程是CPU密集型运算，执行优先级会降低；数值越小，表明进程是I/O密集型运算，执行优先级会提高

STIME：进程启动的时间

TTY：完整的终端名称

TIME：CPU时间

CMD：启动进程所用的命令和参数

3. 终止进程

若是某个进程执行一半需要停止时，或是已消了很大的系统资源时，此时可以考虑停止该进程。使用kill命令来完成此项任务。

  • kill [选项] 进程号，通过进程号杀死/终止进程
  • killall 进程名称，通过进程名称杀死进程，也支持通配符，这在系统因负载过大而变得很慢时很有用
  • -9表示强迫进程立即停止，一般用这个
