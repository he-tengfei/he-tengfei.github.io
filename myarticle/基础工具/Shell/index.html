一、Shell简介

1. 为什么要学习Shell编程

  • Linux运维工程师在进行服务器集群管理时，需要编写Shell程序来进行服务器管理
  • 对于JavaEE和Python程序员来说，工作的需要，你的老大会要求你编写一些Shell 脚本进行程序或者是服务器的维护，比如编写一个定时备份数据库的脚本
  • 能够编写简单的shell脚本用于大数据集群的管理（启动、停止等）
  • 能够大概看懂运维编写的脚本，能看懂50%就行，毕竟这是运维的专业，咱们只是业余

2. Shell是什么

Shell是一个命令行解释器，它接收应用程序/用户命令，然后调用操作系统内核。Shell还是一个功能相当强大的编程语言，易编写、易调试、灵活性强(只有灵活是真的)。

如上图，外层应用程序比如鼠标、键盘等，通过shell进行命令解析，然后调用Linux内核，内核通过驱动程序驱动硬件进行工作。硬件想要工作必须要有驱动程序驱动它。

/bin/bash就好比jvm解析java代码一样，它用来解析shell脚本

二、Shell脚本的入门

编写第一个shell脚本helloworld.sh

脚本格式要求：

  • 脚本内容以**#!/bin/bash**开头，用来指定解析器是谁(不写一般也能执行，但是特定情况下就不能执行了)
  • 后缀名.sh(Linux没有后缀这个概念，这里是约定俗成)
  • 脚本需要有可执行权限

常用的两种执行方式：

  • 赋予执行权限，./helloworld.sh直接执行，或者使用绝对路径/opt/module/helloworld.sh执行
  • 不用赋予执行权限，用解析器+脚本执行：sh helloworld.sh
  • 用. 执行： . helloworld.sh

思考：

为什么给helloworld.sh加过执行权限了，直接输入helloworld.sh却不能执行呢？

因为直接输入脚本名称，Linux不知道去哪儿找，因为不知道路径。为什么直接输入ll就能找到这个命令呢？因为ll在/bin目录下，而/bin目录是在环境变量PATH里的(/bin是软链到/usr/bin的)。如果不输入路径，Linux只会查找环境变量PATH里的目录，找不到就报错。那如果把某个目录配置到环境变量里，然后把shell脚本放里面，是不是就能直接运行了呢？是

父shell与子shell：

通过bash解析的时候，会打开一个子shell，shell脚本在里面执行，变量A也在里面定义，当脚本执行完，子shell被回收，子shell里的变量A也会被回收，所以输出$A为空。但是直接用. 的方式不会打开子shell，就在当前shell里执行。把xshell的窗口关掉，另开一个窗口也相当于另开一个shell，也输出不了$A。

开子shell与不开子shell的区别就在于，环境变量的继承关系，如在子shell中设置的当前变量，父shell是不可见的。

三、Shell的变量

Shell中的变量分为系统变量、自定义变量和特殊变量

1. 系统变量

  • 常用系统变量：$HOME、$PWD、$SHELL、$USER等等，比如：echo $HOME
  • 显示当前shell中所有变量：set

2. 自定义变量

2.1 变量的声明

  • 定义普通变量：变量名=值
  • 撤销普通变量：unset 变量
  • 声明静态变量：readonly 变量，注意静态变量不能unset不能重新赋值

变量声明规范：

  • 变量名称可以由字母、数字和下划线组成，但是不能以数字开头，变量名建议大写
  • 等号两侧不能有空格
  • 在bash中，变量默认类型都是字符串类型，无法直接进行数值运算
  • 变量的值如果有空格，需要使用双引号或单引号括起来
  • shell里不要乱加引号和空格，双引号会解析，单引号不会解析
  • 注意所有普通变量和静态变量，都只在当前shell里存在，当前shell被回收的话变量也会被回收，除非提升为全局变量

2.2 将命令的返回值赋给变量

  • A=`date`，反引号，把命令执行结果返回给变量A
  • A=$(date)，等价于反引号

3. 设置环境变量

-
export 变量名=变量值，将shell变量输出为环境变量/全局变量

  • source 配置文件，让修改后的配置信息立即生效
  • echo $变量名，查询环境变量的值

4. 特殊变量(重要)

又叫位置参数变量

当我们执行一个shell脚本时，如果希望获取到命令行的参数信息，就可以使用到位置参数变量

-
$n：n为数字，$0代表命令本身，$1-$9代表第一到第九个参数，十以上的参数需要用大括号包含，如${10}

  • $*：这个变量代表命令行中所有的参数，$*把所有的参数看成一个整体
  • $@：这个变量也代表命令行中所有的参数，不过$@把每个参数区分对待，用for循环可看出区别
  • $#：获取命令行中所有参数的个数，常用于循环
  • $？：最后一次执行的命令的返回状态。如果这个变量的值为0，证明上一个命令正确执行；如果这个变量的值为非0（具体是哪个数，由命令自己来决定），则证明上一个命令执行不正确了，除了获取函数返回值的情况

四、运算符

基本语法有三种

  • $((运算式))，不推荐，比如T1=$(((2+3)*4))，看起来比较乱，不过运算符里是可以写空格的
  • **$[运算式]**，极其推荐！比如T2=$[(2+3)*4]
  • expr表达式，禁止用！

五、条件判断

1. 语法

  • [ condition ]，注意condition前后要有空格
  • 条件非空即为true，可使用$?验证，比如[ atguigu ]返回true，[ ]返回false
  • [ condition ]&& echo OK | echo notok，条件满足，执行后面的语句？？？

2. 常用判断语句

  • 字符串比较：=
  • 整数比较：-lt小于；-le小于等于；-eq等于；-gt大于；-ge大于等于；-ne不等于 (less greater equal than no)
  • 按照文件权限进行判断：-r；-w；-x
  • 按照文件类型进行判断：-f文件存在并且是一个常规的文件；-e文件存在；-d文件存在并是一个目录

八、函数

shell编程和其它编程语言一样，有系统函数，也可以自定义函数。系统函数中，我们这里就介绍两个。

1. 系统函数

basename [pathname/string] [suffix]：用于返回完整路径最后/之后的部分，常用于获取文件名，suffix为后缀，如果被指定，返回结果会同时去掉后缀

dirname [文件绝对路径]：用于返回文件所在路径，也就是最后/之前的部分

2. 自定义函数

极少人使用，大数据不需要了解，能看懂shell脚本即可

  • 必须在调用函数地方之前，先声明函数，shell脚本是逐行运行，不会像其它语言一样先编译

  • 通过return获得函数返回值的方法比较麻烦而且几乎用不到，详见百度

九、shell工具(重点)

1. cut

cut的工作就是“剪”，具体的说就是在文件中负责剪切数据用的。cut 命令从文件的每一行剪切字节、字符和字段并将这些字节、字符和字段输出。cut并不是在原文件上操作，而是逐行读取然后处理，之后再把结果输出到控制台。

cut [选项参数] filename

选项参数 功能
-f 列号，提取第几列
-d 分隔符，按照指定分隔符分割列，默认分隔符是制表符
-c 指定具体的字符，这个用的很少

2. awk(重点)

一个强大的文本分析工具，把文件逐行的读入，以空格为默认分隔符将每行切片，切开的部分再进行分析处理。cut能做的它都能做，而且比它更强大。

awk [选项参数] ‘pattern1{action1} pattern2{action2}…’ filename

pattern：表示AWK在数据中要查找的内容，就是匹配规则
action：在找到匹配内容时所执行的一系列命令，匹配不到就不执行，往后走

选项参数 功能
-F 指定分隔符
-v 赋值一个用户定义变量

awk的内置变量

变量 说明
FILENME 文件名
NR 已读的记录数(行数)
NF 浏览记录的域的个数（切割后，列的个数）
BEGIN 开始
END 结尾

3. sort

sort命令是将文件进行排序，并将排序结果输出。

**sort (选项) (参数)**，这里参数就是指要排序的文件列表

选项 说明
-n 依照数值的大小排序
-r 以相反的顺序来排序
-t 设置排序时所用的分隔字符
-k 指定需要排序的列

4. wc

wc命令用来计算数字，可以计算文件的Byte数、字数或是列数

wc [选项参数] filename

选项参数 功能
-l 统计文件行数
-w 统计文件的单词数
-m 统计文件的字符数
-c 统计文件的字节数

英文每个字符就占一个字节，中文看编码格式，比如utf-8就占两个字节
