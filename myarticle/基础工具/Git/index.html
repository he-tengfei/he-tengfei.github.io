一、Git概述

官网：https://git-scm.com/

Git 是一个免费的、开源的分布式版本控制系统，可以快速高效地处理从小型到大型的各种项目。

Git 易于学习，占地面积小，性能极快。 它具有廉价的本地库(可以放磁盘)，方便的暂存区域和多个工作流分支等特性。其性能优于 Subversion、CVS、Perforce 和 ClearCase 等版本控制工具。

1. 何为版本控制

版本控制是一种记录文件内容变化，以便将来查阅特定版本修订情况的系统。

版本控制其实最重要的是可以记录文件修改历史记录，从而让用户能够查看历史版本， 方便版本切换。

为什么需要版本控制？从个人开发过渡到团队协作。

小红和小蓝分别修改了代码，小红提交后如果小蓝再提交，就会把小红的覆盖掉，如何把两个人修改的结果都保留并合并呢？

2. 版本控制工具

2.1 集中式版本控制工具

CVS、SVN(Subversion)、VSS……

集中化的版本控制系统诸如 CVS、SVN 等，都有一个单一的集中管理的服务器，保存所有文件的修订版本，而协同工作的人们都通过客户端连到这台服务器，取出最新的文件或者提交更新。多年以来，这已成为版本控制系统的标准做法。

这种做法带来了许多好处，每个人都可以在一定程度上看到项目中的其他人正在做些什么。而管理员也可以轻松掌控每个开发者的权限，并且管理一个集中化的版本控制系统，要远比在各个客户端上维护本地数据库来得轻松容易。

事分两面，有好有坏。这么做显而易见的缺点是中央服务器的单点故障。如果服务器宕机一小时，那么在这一小时内，谁都无法提交更新，也就无法协同工作。

2.2 分布式版本控制工具

Git、Mercurial、Bazaar、Darcs……

像 Git 这种分布式版本控制工具，客户端提取的不是最新版本的文件快照，而是把代码从远程库完整地镜像下来（本地库）。这样任何一处协同工作用的文件发生故障，事后都可以用其他客户端的本地仓库进行恢复。因为每个客户端的每一次文件提取操作，实际上都是一次对整个文件仓库的完整备份。

分布式的版本控制系统出现之后,解决了集中式版本控制系统的缺陷:

  • 服务器断网的情况下也可以进行开发（因为版本控制是在本地进行的，只是无法推送）
  • 每个客户端保存的也都是整个完整的项目（包含历史记录，更加安全）

3. Git工作机制

工作区指的是你代码存放的磁盘的目录位置，写好代码后add进暂存区，暂存区只是临时存储，也可以随时删除，此时依然没有生产历史版本，只有把暂存区的代码提交到本地库，才会生产历史版本，从此这个代码就再也删不掉了，除非删库跑路

4. Git 和代码托管中心

代码托管中心是基于网络服务器的远程代码仓库，一般我们简单称为远程库。

  • 局域网：GitLab
  • 互联网：GitHub（外网）、Gitee 码云（国内网站）

四、Git分支操作

1. 什么是分支？

在版本控制过程中，同时推进多个任务，为每个任务，我们就可以创建每个任务的单独分支。使用分支意味着程序员可以把自己的工作从开发主线上分离开来，开发自己分支的时候，不会影响主线分支的运行。

比如下图，master分支正在生产环境上运行，要增加新功能，在代码修改期间又不能对原来的版本造成影响，另开个modify分支。modify分支完成后与master分支进行合并，合并后发现异常卡顿，于是赶紧另开hot-fix分支对其进行紧急修复，修复完与master分支合并。

  • 分支底层其实也是指针的引用。
  • 分支可以同时并行推进多个功能开发，提高开发效率
  • 如果某一个分支开发失败，不会对其他分支有任何影响，失败的分支删除重新开始即可

注意：push是将本地库代码推送到远程库，如果本地库代码跟远程库代码版本不一致，push的操作是会被拒绝的。也就是说，要想push成功，一定要保证本地库的版本要比远程库的版本高！因此一个成熟的程序员在动手改本地代码之前，一定会先检查下远程库跟本地代码的区别！如果本地的代码版本已经落后，切记要先pull拉取一下远程库的代码，将本地代码更新到最新以后，然后再修改，提交，推送！

2. 报错

错误1

创建仓库时，勾选了“使用Readme文件初始化这个仓库”这个操作，创建了一个README文件并配置添加了忽略文件，此时我们的仓库就有了README.md和.gitignore文件。之后把本地推送到远程时，两端都是有内容的，但是这两份内容并没有联系，当我们推送到远程或者从远程拉取内容时，都会有没有被跟踪的内容，于是你看git报的详细错误中总是会让你先拉取再推送，但是拉取总是失败。所以新建仓库的时候千万不要勾选readme。

补救办法：先rebase，然后再push

这个报错是之后再推送时发生的，原因居然是没写commit！解决办法就是把上面的步骤，除了init外全走一遍，只会push更改过的内容。
